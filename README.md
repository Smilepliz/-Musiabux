# Musiabux

## Установка в один клик (для стороннего пользователя)

Сторонний пользователь может начать работу **одним из трёх способов** — без настроек и лишних шагов.

---

### 1️⃣ Самый простой — в браузере (ничего не ставить)

Откройте ссылку — приложение сразу откроется в браузере:

👉 **[Открыть Musiabux в браузере](https://smilepliz.github.io/-Musiabux/)**

Подойдёт для любого компьютера и телефона. Можно сохранить в закладки или отправить ссылку кому угодно.

---

### 2️⃣ Один клик — скачать exe и запустить

1. Скачайте программу одним кликом: **[Musiabux.exe](https://github.com/Smilepliz/-Musiabux/raw/main/Musiabux.exe)**
2. Запустите скачанный файл — установка не требуется, работает без интернета.

Альтернатива: [Releases](https://github.com/Smilepliz/-Musiabux/releases) — там всегда актуальная сборка.

---

### 3️⃣ Из исходников — один клик после скачивания папки

Если вы скачали репозиторий (ZIP или клон):

1. Установите [Node.js](https://nodejs.org) (если ещё не установлен).
2. Дважды нажмите **Запуск.bat** — зависимости установятся сами, приложение откроется в браузере.

Чтобы собрать свой exe на этом компьютере — дважды нажмите **Собрать exe.bat**.

---

## Для владельца репозитория

- **GitHub Pages:** репозиторий → **Settings** → **Pages** → **Build and deployment** → **Source**: выберите **GitHub Actions**. Без этого проверка «Deploy to GitHub Pages» в CI будет падать.
- Exe собирается автоматически при пуше в `main` и попадает в [Releases](https://github.com/Smilepliz/-Musiabux/releases) и в корень репозитория.
- Если падает **Build EXE**: откройте вкладку Actions → выберите упавший запуск → откройте job **build** и посмотрите, на каком шаге ошибка (часто это `npm run build:exe` или шаг с Release).
